# 模型说明文档

## 一、任务定义

本项目的核心机器学习任务是：**根据气象条件预测自然光的相对光谱曲线**。

- **输入（X）**：8 个气象特征（hour, cloud_cover, humidity, temperature, precipitation, solar_altitude, outdoor_lux, weather）
- **输出（Y）**：41 维相对光谱（wavelength_380 ~ wavelength_780，每 10nm 一个采样点）

这是一个**多输出回归问题**——输入一组气象参数，输出一条完整的光谱曲线。

---

## 二、特征工程

### 2.1 输入特征预处理

| 特征 | 类型 | 预处理方式 | 说明 |
|------|------|-----------|------|
| hour | 连续 | StandardScaler | 小时（7–19） |
| cloud_cover | 连续 | StandardScaler | 云量（0–1） |
| humidity | 连续 | StandardScaler | 相对湿度（0–1） |
| temperature | 连续 | StandardScaler | 气温（℃） |
| precipitation | 连续 | StandardScaler | 降水量（mm/h） |
| solar_altitude | 连续 | StandardScaler | 太阳高度角（°） |
| outdoor_lux | 连续 | StandardScaler | 室外照度（lux） |
| weather | 类别 | OneHotEncoder | 天气分类 → 4 列哑变量 |

连续特征用 `StandardScaler` 标准化（均值 0、方差 1），消除量纲差异。类别特征 `weather` 用 `OneHotEncoder` 编码为 4 列（晴/多云/阴/雨），因为天气之间没有自然的大小顺序关系。

### 2.2 目标变量降维（PCA）

直接预测 41 维光谱存在两个问题：

1. **维度高、样本相对少**：5,664 个样本预测 41 个输出，容易过拟合
2. **相邻波长高度相关**：光谱是连续曲线，380nm 和 390nm 的值几乎总是同增同减，信息冗余

因此对目标变量做 PCA 降维：

```
41 维原始光谱
    ↓ StandardScaler（标准化）
41 维标准化光谱
    ↓ PCA（保留 5 个主成分）
5 维 PCA 成分
```

5 个主成分通常能解释 95% 以上的方差。训练时模型学习 `X → 5 维 PCA 成分`，预测时通过 `pca.inverse_transform()` + `StandardScaler.inverse_transform()` 还原为 41 维光谱。

## 三、模型选型

### 3.1 选用的 4 个模型

| 模型 | 类型 | sklearn 实现 |
|------|------|-------------|
| Linear Regression | 线性回归 | `LinearRegression()` |
| KNN Regression | K 近邻回归 | `KNeighborsRegressor(n_neighbors=12, weights="distance")` |
| Decision Tree | 决策树回归 | `DecisionTreeRegressor(max_depth=14)` |
| Random Forest | 随机森林回归 | `RandomForestRegressor(n_estimators=180, max_depth=18, min_samples_leaf=2)` |

### 3.2 为什么选这 4 个模型

选择覆盖四种不同的学习范式，从简单到复杂逐一对比：

| 模型 | 选型理由 |
|------|---------|
| **Linear Regression** | 作为基线模型。如果线性关系就能解释大部分变化，说明问题简单；如果效果差，说明气象特征和光谱之间存在非线性关系，需要更复杂的模型 |
| **KNN Regression** | 实例学习方法，不需要显式建模。思路是"找历史上天气条件最相似的日子，用它们的光谱做参考"。适合捕捉局部模式 |
| **Decision Tree** | 非线性模型，通过 if-else 规则切分特征空间。能捕捉特征之间的交互关系（如"云量高且太阳高度角低时光谱偏暗"），但单棵树容易过拟合 |
| **Random Forest** | 集成学习方法，180 棵决策树投票。保留了决策树的非线性能力，同时通过 bagging 和特征随机化大幅降低过拟合风险，是结构化数据回归的常用最优选择 |

### 3.3 为什么 Random Forest 最终胜出

1. **天气→光谱是非线性映射**：云量从 0.2→0.4 的光谱变化量，和从 0.8→1.0 的变化量不同；湿度的影响还依赖太阳高度角。线性回归无法建模这种交互关系。

2. **数据量适中**：5,664 个样本、8 个特征——这是随机森林的舒适区。样本太少会过拟合，特征太多会变慢，当前规模正好。

3. **光谱物理规律性强**：给定相同的天气条件，光谱形态基本确定（低噪声），随机森林能充分学到这个确定性映射。

4. **抗过拟合**：180 棵树集成后，个体偏差互相抵消，泛化能力强于单棵决策树。

## 四、模型超参数详解

### 4.1 当前超参数设定

| 模型 | 超参数 | 当前值 | 含义 |
|------|--------|--------|------|
| KNN | n_neighbors | 12 | 参考最近的 12 个邻居 |
| KNN | weights | "distance" | 距离越近的邻居投票权重越大 |
| Decision Tree | max_depth | 14 | 树的最大深度（最多 14 层 if-else） |
| Random Forest | n_estimators | 180 | 集成 180 棵决策树 |
| Random Forest | max_depth | 18 | 每棵树的最大深度 |
| Random Forest | min_samples_leaf | 2 | 每个叶子节点最少包含 2 个样本 |
| Random Forest | random_state | 42 | 随机种子，保证可复现 |
| Random Forest | n_jobs | -1 | 使用全部 CPU 核心并行训练 |

### 4.2 超参数选择依据

当前超参数是通过手动试参确定的，没有使用自动化搜索工具（如 GridSearchCV）。选择逻辑如下：

**n_estimators（树的数量）**：
- 太少（<50）：预测不稳定，不同随机种子结果差异大
- 太多（>300）：训练变慢，但预测提升边际递减
- 180：效果和速度的平衡点

**max_depth（树的深度）**：
- 太浅（<8）：欠拟合，无法捕捉复杂的光谱模式
- 太深（>25）：过拟合，训练集表现好但测试集变差
- 18：每棵树最多做 18 次特征切分，足以学到天气→光谱的非线性关系

**min_samples_leaf（叶子最小样本数）**：
- 设为 1：允许树创建只包含 1 个样本的叶子，该叶子的输出完全由那一个样本决定，极易过拟合
- 设为 2：要求每个叶子至少看到 2 个训练样本，输出更平滑

## 五、Random Forest 调优方法

### 5.1 需要调优的超参数

| 超参数 | 类型 | 搜索范围 | 调优方向 |
|--------|------|---------|---------|
| n_estimators | 整数 | 100–300 | 增大→更稳定但更慢 |
| max_depth | 整数或 None | 10–30 或不限 | 增大→更强拟合但可能过拟合 |
| min_samples_leaf | 整数 | 1–10 | 增大→更强正则化、更平滑 |
| min_samples_split | 整数 | 2–20 | 增大→更保守的分裂 |
| max_features | float 或 int | 0.3–1.0 或 "sqrt" | 减小→树之间更独立、抗过拟合更强 |

### 5.2 方法一：GridSearchCV（网格搜索）

穷举所有参数组合，适合参数空间较小的情况。

```python
from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestRegressor

param_grid = {
    "n_estimators": [120, 180, 240],
    "max_depth": [14, 18, 22, None],
    "min_samples_leaf": [1, 2, 4],
}

grid = GridSearchCV(
    estimator=RandomForestRegressor(random_state=42, n_jobs=-1),
    param_grid=param_grid,
    cv=5,                  # 5 折交叉验证
    scoring="neg_root_mean_squared_error",  # 以 RMSE 为评价指标
    n_jobs=-1,
    verbose=1,
)
grid.fit(X_train, y_train_pca)
print("最佳参数:", grid.best_params_)
print("最佳 RMSE:", -grid.best_score_)
```

本例共 3×4×3 = 36 种组合，每种做 5 折交叉验证，共训练 180 个模型。对 Random Forest 来说耗时可接受。

### 5.3 方法二：RandomizedSearchCV（随机搜索）

在参数空间中随机采样，适合参数空间较大或计算资源有限的情况。

```python
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint, uniform

param_distributions = {
    "n_estimators": randint(100, 300),
    "max_depth": randint(10, 30),
    "min_samples_leaf": randint(1, 10),
    "min_samples_split": randint(2, 20),
    "max_features": uniform(0.3, 0.7),  # 0.3 ~ 1.0
}

search = RandomizedSearchCV(
    estimator=RandomForestRegressor(random_state=42, n_jobs=-1),
    param_distributions=param_distributions,
    n_iter=50,             # 随机采样 50 种组合
    cv=5,
    scoring="neg_root_mean_squared_error",
    n_jobs=-1,
    random_state=42,
    verbose=1,
)
search.fit(X_train, y_train_pca)
print("最佳参数:", search.best_params_)
print("最佳 RMSE:", -search.best_score_)
```

50 次采样 × 5 折 = 250 个模型，比网格搜索覆盖面更广，但单次运行更随机。

### 5.4 方法三：手动逐步调优

从当前参数出发，逐个调整，每次只改一个参数观察效果：

1. **先固定其他参数，调 n_estimators**：试 100、150、180、200、250，观察 RMSE 变化
2. **再调 max_depth**：试 12、15、18、22、None（不限），观察是否过拟合
3. **最后调 min_samples_leaf**：试 1、2、4、8，观察测试集 RMSE

这种方法计算量最小，适合理解每个参数的影响。

### 5.5 调优评价标准

| 指标 | 用途 |
|------|------|
| **RMSE（交叉验证）** | 主要调优依据，选择 RMSE 最低的参数组合 |
| **R²** | 辅助参考，确保模型解释力没有退化 |
| **训练集 vs 测试集 RMSE 差距** | 判断是否过拟合。差距大→过拟合，需要更强正则化 |
| **OOB Score** | Random Forest 自带的袋外评估，无需额外划分验证集 |

Random Forest 的 `oob_score=True` 参数可以启用袋外评估：

```python
rf = RandomForestRegressor(
    n_estimators=180, max_depth=18, min_samples_leaf=2,
    oob_score=True, random_state=42, n_jobs=-1
)
rf.fit(X_train, y_train_pca)
print("OOB R²:", rf.oob_score_)
```

## 六、评价指标说明

### 6.1 三个指标的含义

| 指标 | 计算方式 | 直观含义 | 好坏标准 |
|------|---------|---------|---------|
| MAE | mean(\|预测 - 真实\|) | 平均每个波长点偏了多远 | 越小越好 |
| RMSE | sqrt(mean((预测 - 真实)²)) | 类似 MAE，但对大误差惩罚更重 | 越小越好 |
| R² | 1 - SS_res / SS_total | 模型解释了百分之多少的光谱变化 | 越接近 1 越好 |

### 6.2 为什么用 RMSE 排序选最优模型

代码中使用 `sort_values(["RMSE", "MAE"])` 选择最优模型，原因：

1. **R² 区分度不够**：4 个模型的 R² 都很高（>0.99），拉不开差距
2. **RMSE 对大误差敏感**：LED 补偿需要的是"预测曲线和真实曲线整体偏差小"，RMSE 正好衡量这个
3. **MAE 作为辅助**：如果 RMSE 高但 MAE 低，说明模型偶尔有大偏差（不稳定）；如果两个都高，说明整体预测能力不足

### 6.3 为什么 R² 能达到 0.9997

1. **物理确定性**：给定相同的天气条件，自然光光谱是确定性的，不存在随机性
2. **光谱内在结构**：41 维光谱有很强的连续性，PCA 只需 5 个成分就能解释 95%+ 方差
3. **Random Forest 适合此规模**：8 个特征、5,664 个样本、非线性关系——恰好是随机森林的最优应用场景

## 七、模型部署

### 7.1 训练结果序列化

训练完成后，整个 `ModelTrainingResult`（包含 Pipeline、PCA、Scaler、测试集预测等）通过 `joblib.dump()` 保存到 `models/spectrum_model.joblib`。

### 7.2 预测流程

```
用户输入天气参数（小时、云量、湿度、温度、降水、太阳高度角、照度、天气类型）
    ↓
Pipeline.predict() → [StandardScaler + OneHotEncoder] → 5 维 PCA 成分
    ↓
pca.inverse_transform() → StandardScaler 逆变换 → 41 维预测光谱
    ↓
normalize_curve() → 归一化到 [0, 1]
    ↓
输出：相对光谱曲线
```

### 7.3 特征重要性

Random Forest 提供 `feature_importances_` 属性，可以输出每个输入特征对预测的贡献度。代码中通过 `_aggregate_feature_importance()` 将 OneHotEncoder 编码后的天气列合并回一个 `weather` 特征，输出 8 个特征的重要性排序。
