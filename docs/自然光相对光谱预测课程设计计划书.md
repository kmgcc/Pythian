## 一、课题名称
**基于机器学习的自然光相对光谱预测与室内照明补偿设计**

## 二、课题背景与主旨
现代人每天有大量时间处在室内环境中。相关研究和资料中常见的估计是，人们约 80%～90% 的时间在室内或封闭空间中度过。因此，室内环境质量已经不只是建筑设计问题，也与学习效率、视觉舒适、生理节律和长期健康体验有关。

在室内环境中，照明是最容易被忽视、但又最常接触的因素之一。传统照明设计往往主要关注“够不够亮”，也就是照度是否达标；近些年则开始进一步关注色温，例如冷白光、暖白光、昼夜节律照明等。但从光本身来看，亮度和色温只是对光照信息的高度压缩。两盏灯即使亮度相同、色温相同，它们的光谱分布也可能完全不同，对视觉感受、颜色还原和生理影响也可能不同。

因此，本课题尝试从“光谱”的角度理解自然光与室内照明。项目不追求科研级或工程级的真实太阳光谱预测，而是基于标准日光光谱和模拟天气数据，构建一个课程设计层面的自然光相对光谱预测与室内照明补偿 Demo。它的重点是用 Python 完成一个完整流程：数据构建、数据分析、PCA 降维、机器学习建模、模型对比、光谱预测、可视化展示和补偿方案设计。

此外，光谱仪设备价格昂贵，若每个家庭都部署实体光谱仪进行实时监测，成本过高且不切实际。通过机器学习模型预测自然光光谱，可以大幅降低硬件成本，使光谱级照明调节更易于普及和应用。

本课题的核心主旨是：
> 在人们长期处于室内环境的现实下，室内照明不应只停留在亮度和色温层面。未来更高质量的照明系统，应该能够理解自然光的光谱变化，并根据自然光条件主动调节人工照明，使室内光环境更接近自然、更舒适、更节能。

## 三、课程设计定位
本项目属于 Python 应用开发课程设计，要求以 Python 语言为基础，结合课程所学知识，设计并实现一个具有数据处理、智能算法和可视化展示的小型应用系统。

本项目将重点体现以下课程内容：
1. **Python 数据处理**
   - 使用 `pandas` 读取、生成和整理数据；
   - 使用 `numpy` 处理光谱矩阵、数值计算和模拟数据生成。
2. **数据可视化**
   - 使用 `matplotlib` 和 `seaborn` 绘制标准光谱曲线、不同天气光谱对比图、PCA 解释方差图、模型预测效果图、补偿前后对比图等。
3. **机器学习基础**
   - 将“天气、时间、太阳高度角等特征 → 自然光相对光谱”视为一个回归预测问题；
   - 理解训练集、测试集、特征、标签、模型、预测、评价指标等基本概念。
4. **PCA 主成分分析**
   - 光谱数据维度较高，例如 380nm～780nm 每 10nm 一个点，共 41 个维度；
   - 使用 PCA 将高维光谱压缩成少量主成分系数；
   - 再用机器学习模型预测 PCA 系数，最后还原完整光谱曲线。
5. **回归模型与模型对比**
   - 对比线性回归、KNN 回归、决策树回归、随机森林回归等模型；
   - 使用 MAE、RMSE、R² 等指标比较模型效果；
   - 选择综合效果较好的模型作为最终预测模型。
6. **特征重要性分析**
   - 使用随机森林模型输出特征重要性；
   - 分析时间、太阳高度角、云量、天气类型等因素对光谱预测的影响。
7. **简单优化思想**
   - 根据目标光谱与预测自然光谱的差异，计算人工照明补偿比例；
   - 假设室内灯具具有多通道光谱可调 LED，通过最小二乘或简单搜索方法生成补偿建议。

## 四、项目目标
本项目计划实现以下目标：
1. 构建一个基于标准光谱与模拟天气条件的自然光相对光谱数据集；

3. 使用 PCA 对光谱数据进行降维，降低模型训练难度；
4. 使用多种机器学习回归模型预测自然光相对光谱；
5. 对不同模型进行评价和对比，选择较优模型；
6. 将预测结果还原为完整光谱曲线；
7. 根据预测自然光谱生成室内照明补偿方案；
8. 完成课程代码演示、课程报告和 PPT 答辩材料。

## 六、数据设计

### 6.1 数据来源说明

由于公开的“天气预报数据 + 实测自然光完整光谱”的数据集较难获取，本项目计划采用以下方式构建课程设计数据集：

1. **基准光谱数据**：优先使用公开的标准太阳光谱数据作为基准，例如 ASTM G173 / AM1.5G 标准太阳光谱，或其他公开标准太阳光谱表。项目只取可见光范围，例如 380nm～780nm，共 41 个波长点（每 10nm 一个采样点）。
2. **天气数据获取**：优先尝试通过天气 API 获取真实天气数据，例如 Open-Meteo（可获取云量、温度、湿度、降水、辐射等字段，适合课程设计）或 OpenWeatherMap（备选，可能需要注册 API Key）。如果 API 调用不稳定或时间不够，则使用本地模拟数据作为 fallback。
3. **模拟样本生成**：以真实/标准光谱为基准，结合天气、云量、太阳高度角、湿度等因素，生成模拟扩展样本。这样比完全随机生成数据更可信，能保证光谱曲线形态具有基本物理参考，也能让课程设计中的机器学习流程更容易解释。
4. **数据集构成**：得到一批不同天气、不同时间条件下的自然光相对光谱样本，将这些样本作为机器学习模型训练和演示的数据集。

报告中需要明确说明：

本项目不声称拥有大规模实测自然光光谱数据。本项目采用真实/标准太阳光谱作为基准，并基于简化天气规则生成课程设计用模拟样本。模拟数据用于验证流程，不代表工程级精确预测。

## 八、核心算法设计

### 8.1 光谱模拟生成

以标准日光光谱（如 ASTM G173 / AM1.5G）作为基础曲线，根据不同天气和时间进行扰动。

初步规则如下：

| 条件 | 模拟规则 |
|------|----------|
| 晴天 | 光谱接近标准日光，整体强度较高 |
| 多云 | 整体强度下降，光谱略微平滑 |
| 阴天 | 强度明显下降，光谱变化幅度减小 |
| 雨天 | 强度更低，整体光谱更平缓 |
| 早晨/傍晚 | 太阳高度角较低，短波蓝光部分衰减更多，光谱偏暖 |
| 中午 | 太阳高度角较高，光谱更接近标准日光 |

这部分的重点是形成“可解释的模拟数据”，而不是追求真实大气物理模型的严谨性。

### 8.2 PCA 光谱降维

原始光谱数据包含多个波长点，维度较高，且相邻波长之间存在明显相关性。因此计划使用 PCA 主成分分析进行降维。

处理流程：
```
原始光谱矩阵 X
        ↓
标准化处理
        ↓
PCA 降维
        ↓
保留前 3～5 个主成分
        ↓
获得每条光谱的 PCA 系数
```

PCA 的作用：
1. 降低数据维度；
2. 保留主要光谱变化趋势；
3. 减少模型直接预测 41 个波长点的难度；
4. 方便结合课程中学习过的降维知识；
5. 通过解释方差图展示主成分保留的信息量。

计划展示：
- PCA 主成分解释方差比例；
- 累计解释方差曲线；
- 原始光谱与 PCA 还原光谱对比。

### 8.3 机器学习模型设计

本项目将光谱预测任务转化为回归任务：

- 输入：天气、时间、太阳高度角、云量、湿度、室外照度等特征
- 输出：光谱 PCA 主成分系数

计划对比以下模型：

| 模型 | 作用 | 选择原因 |
|------|------|----------|
| Linear Regression | 线性回归基准模型 | 简单，作为对照组 |
| KNeighborsRegressor | KNN 回归 | 体现课程中 KNN 思想 |
| DecisionTreeRegressor | 决策树回归 | 结构直观，便于解释 |
| RandomForestRegressor | 随机森林回归 | 稳定性较好，适合非线性关系 |
| MLPRegressor（可选） | 多层感知机 | 体现神经网络基础，作为扩展 |

由于输出是多个 PCA 系数，可以使用多输出回归形式。最终将根据模型在测试集上的 MAE、RMSE、R² 等指标进行对比，选择效果最好的模型作为最终模型。

**为什么使用机器学习模型而不是直接使用大模型预测？**

LLM 适合辅助理解、资料整理、报告润色和代码解释，但不适合直接承担数值预测核心。自然光光谱预测是连续数值回归问题，需要稳定、可复现、可评价的模型。自己训练机器学习模型的价值包括：
- 准确性：可以用 MAE、RMSE、R² 等指标评价结果
- 可控性：模型输入、输出、训练数据和参数都可检查
- 可复现性：同样数据和代码可以重复运行
- 隐私与部署：未来如果接入室内传感器或用户位置数据，本地模型更容易保护隐私
- 效率：传统机器学习模型推理速度快，适合实时照明调节
- 课程适配：能体现 Python、PCA、sklearn、回归模型、多模型对比等课程内容

大模型参与方式主要是过程实施辅助，例如：
- 辅助查找光谱和天气 API 资料
- 辅助理解 ASTM G173、天气字段、光谱补偿等概念
- 辅助检查代码注释、报告语言和 PPT 表达
- 辅助解释模型结果

但不替代核心数据生成、模型训练和结果评价。

### 8.4 模型评价指标

计划使用以下指标：

| 指标 | 含义 |
|------|------|
| MAE | 平均绝对误差，反映平均预测偏差 |
| RMSE | 均方根误差，对较大误差更敏感 |
| R² | 决定系数，反映模型解释能力 |
| 光谱曲线对比 | 通过图像直观展示预测效果 |

最终展示方式：
1. 模型评价指标表；
2. 不同模型误差对比柱状图；
3. 预测光谱与模拟真实光谱曲线对比图。

### 8.5 光谱还原

模型预测出的不是原始 41 维光谱，而是 PCA 主成分系数。预测完成后，使用 PCA 的 inverse_transform() 方法将主成分系数还原为完整光谱曲线。

流程：
```
模型预测 PCA 系数
        ↓
PCA 逆变换
        ↓
还原为 380nm～780nm 光谱曲线
        ↓
与模拟真实光谱对比
```

这样可以把机器学习结果转化为直观的光谱图，便于代码演示和 PPT 展示。

### 8.6 室内多通道 LED 照明补偿设计

本项目假设室内灯具为多通道光谱可调 LED，例如七通道混光系统：

| 通道 | 说明 |
|------|------|
| Channel 1 | 深蓝/蓝光通道 |
| Channel 2 | 青光通道 |
| Channel 3 | 绿光通道 |
| Channel 4 | 琥珀光通道 |
| Channel 5 | 红光通道 |
| Channel 6 | 暖白通道 |
| Channel 7 | 冷白通道 |

补偿思路：

目标光谱 - 当前预测自然光谱 = 需要人工补偿的光谱

然后通过最小二乘或简单搜索，计算各 LED 通道的推荐比例，使“自然光 + 人工补偿光”尽量接近目标光谱。

输出内容包括：
1. 当前预测自然光谱；
2. 目标光谱；
3. 推荐 LED 通道比例；
4. 补偿后光谱曲线；
5. 补偿前后误差对比。

**行业前沿说明：**
高质量照明正在从亮度/色温控制走向光谱控制。多通道 LED 混光可以比 RGB 更细致地调节光谱，避免窄峰光谱造成的显色问题。课程设计中使用七通道模型，是为了表达“光谱级照明补偿”的思路，而不是做真实硬件控制。

我们为什么做这个题？因为人长期在室内，而照明质量对室内体验很重要。

为什么关注光谱？因为亮度和色温不能完整描述光照，同色温、同亮度下光谱可能不同。

我们怎么做？用标准日光光谱和天气规则生成模拟数据，用 PCA 降维，再用机器学习预测光谱主成分。

怎么证明模型有效？对比线性回归、KNN、决策树、随机森林等模型，用 MAE、RMSE、R² 评价，选择效果较好的模型。

有什么应用？根据预测自然光谱，计算人工多通道 LED 补偿比例，使室内合成光谱更接近目标自然光谱。

项目有什么不足？数据是模拟数据，没有真实光谱仪采集；补偿方案没有接入硬件；未来可结合天气 API、光谱传感器和智能灯进一步完善。

## 十九、项目不足与改进方向

本项目也存在明显限制，需要在报告和答辩中主动说明。

### 19.1 不足
1. 数据主要为模拟数据，缺少真实光谱仪长期采集数据；
2. 天气和大气因素被简化，未充分考虑空气污染、气溶胶、水汽、建筑遮挡等复杂因素；
