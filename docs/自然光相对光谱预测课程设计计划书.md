# 《基于真实天光数据的自然光光谱估计与室内照明补偿设计》课程设计实施计划书

> [!NOTE]
> 本计划书与项目正式版本保持同步。项目统一使用真实公开实测天光光谱数据（SKYSPECTRA，Zenodo record 8147546），并通过 Open-Meteo 历史天气 API 按地点和时间对齐历史气象特征；数据清洗、建模与可视化全部基于真实数据运行。

## 一、课题名称
**基于真实天光数据的自然光光谱估计与室内照明补偿设计**

## 二、课题背景与主旨
现代人每天有大量时间处在室内环境中。相关研究和资料中常见的估计是，人们约 80%～90% 的时间在室内或封闭空间中度过。因此，室内环境质量已经不只是建筑设计问题，也与学习效率、视觉舒适、生理节律和长期健康体验有关。

在室内环境中，照明是最容易被忽视、但又最常接触的因素之一。传统照明设计往往主要关注"够不够亮"，也就是照度是否达标；近些年则开始进一步关注色温，例如冷白光、暖白光、昼夜节律照明等。但从光本身来看，亮度和色温只是对光照信息的高度压缩。两盏灯即使亮度相同、色温相同，它们的光谱分布也可能完全不同，对视觉感受、颜色还原和生理影响也可能不同。

本课题尝试从"光谱"的角度理解自然光与室内照明。项目使用公开实测天光光谱数据，结合天气、时间、地点、太阳位置等低成本环境特征，训练机器学习模型估计自然光相对光谱，并进一步用于室内照明补偿设计。重点是用 Python 完成一条完整流程：真实数据获取、数据分析、PCA 降维、机器学习建模、模型对比、光谱估计、可视化展示和补偿方案设计。项目不试图完整解决真实建筑照明中的复杂问题，属于课程设计层面的前瞻性实验。

此外，光谱仪设备价格昂贵，若每个家庭都部署实体光谱仪进行实时监测，成本过高且不切实际。通过机器学习模型估计自然光光谱，可以大幅降低硬件成本，使光谱级照明调节更易于普及和应用。

本课题的核心主旨是：
> 在人们长期处于室内环境的现实下，室内照明不应只停留在亮度和色温层面。未来更高质量的照明系统，应该能够理解自然光的光谱变化，并根据自然光条件主动调节人工照明，使室内光环境更接近自然、更舒适、更节能。

## 三、课程设计定位
本项目属于 Python 应用开发课程设计，要求以 Python 语言为基础，结合课程所学知识，设计并实现一个具有数据处理、智能算法和可视化展示的小型应用系统。

本项目将重点体现以下课程内容：
1. **Python 数据处理**
   - 使用 `pandas` 读取、清洗、合并和整理多个真实数据文件；
   - 使用 `numpy` 处理光谱矩阵、线性插值重采样和数值计算。
2. **数据可视化**
   - 使用 `matplotlib` 和 `seaborn` 绘制光谱曲线、不同天气光谱对比图、PCA 解释方差图、模型对比图、补偿前后对比图等。
3. **机器学习基础**
   - 将"天气、时间、地点、太阳位置等特征 → 自然光相对光谱"视为一个回归预测问题；
   - 理解训练集、测试集、特征、标签、模型、预测、评价指标等基本概念。
4. **特征工程**
   - 类别特征独热编码（天气类别、天空状况、站点）；
   - 数值特征标准化（KNN、MLP、线性模型都依赖）；
   - 光谱归一化：每条光谱按自身最大值归一化为相对光谱。
5. **PCA 主成分分析**
   - 光谱数据维度较高，380nm～780nm 每 10nm 一个点，共 41 个维度；
   - 使用 PCA 将高维光谱压缩成少量主成分系数，并用解释方差选择主成分数量；
   - 再用机器学习模型预测 PCA 系数，最后 inverse_transform 还原完整光谱曲线。
6. **回归模型与模型对比**
   - 对比线性回归、KNN 回归、决策树回归、随机森林回归、MLP 回归五个模型；
   - 使用 MAE、RMSE、R²、训练时间、预测时间统一评价；
   - 由真实指标选择最终模型，不预设答案。
7. **简单优化思想**
   - 根据目标光谱与预测自然光谱的差异，用非负最小二乘计算七通道 LED 补偿比例。

## 四、项目目标
本项目计划实现以下目标：
1. 用 Python 代码下载并整理真实公开天光光谱数据，结合元数据和历史天气构建建模数据集；
2. 使用 Python 对数据进行读取、清洗、合并、插值和可视化；
3. 使用 PCA 对光谱数据进行降维，降低模型训练难度；
4. 使用五种机器学习回归模型估计自然光相对光谱；
5. 对不同模型进行统一评价和对比，选择较优模型；
6. 将预测结果还原为完整光谱曲线；
7. 根据预测自然光谱生成室内照明补偿方案；
8. 完成课程代码演示、课程报告和 PPT 答辩材料。

## 六、数据设计

### 6.1 数据来源说明

本项目全部使用公开真实数据，由 Python 代码联网获取并本地缓存：

1. **实测天光光谱数据**：SKYSPECTRA 数据集（Zenodo record 8147546），`spectral_horizontal_irradiance.csv` 提供光谱测量长表（约 78MB），元数据文件提供地点坐标、时间、天空状况、太阳位置和室外照度。光谱取可见光范围 380nm～780nm，重采样为每 10nm 一个点，共 41 个波长点。
2. **历史天气数据**：Open-Meteo Historical Weather API，按观测站经纬度和日期范围批量查询云量、湿度、温度、降水，对齐到光谱测量的最近整点。
3. **实测 LED 光谱数据**：Harald Brendel 公开 LED SPD 数据，用于构造补偿环节的七通道 LED 光谱，不参与估计模型训练。

报告中需要明确说明：

本项目使用公开实测天光光谱数据作为主要数据来源，并结合数据集元数据与历史天气特征构建模型输入。天气数据为按地点和时间对齐的历史天气特征，不等同于现场同步气象测量。

### 6.2 数据字段设计

最终数据表 `real_spectrum_weather_dataset.csv` 字段如下：

| 字段名 | 含义 |
|--------|------|
| sample_id | 样本编号 |
| date / hour / month | 日期、小时、月份 |
| location_code / station_name | 观测站编号与站名（取自元数据） |
| latitude / longitude | 站点坐标 |
| weather | 天气类别（晴、多云、阴、雨，由云量和降水划分） |
| sky_condition | 数据集自带天空状况（clear/overcast/other） |
| cloud_cover / humidity / temperature / precipitation | 历史天气特征 |
| solar_altitude / solar_azimuth | 太阳高度角、方位角 |
| outdoor_lux | 实测室外水平照度 |
| wavelength_380 ~ wavelength_780 | 41 个波长点的光谱强度 |

室内场景与目标照度由用户在补偿环节选择，不写入数据集。

### 6.3 数据量

实际匹配成功的样本为 5664 条（法国沃昂夫兰站 5636 条、新加坡站 28 条），时间范围 2016-03-16 至 2018-07-28。

## 八、核心算法设计

### 8.1 数据处理

原始光谱为长表，需要清洗（去缺失、去负值、滤全 0 光谱）、透视为宽表，再用 `numpy.interp` 线性插值重采样到统一波长网格。元数据按"站点 + 时间戳"合并，天气按"站点 + 最近整点"对齐。每条光谱按自身最大值归一化为相对光谱，模型只学习光谱形状，照度由 outdoor_lux 特征单独表达。

### 8.2 PCA 光谱降维

原始光谱 41 维，相邻波长相关性强，计划使用 PCA 降维：

```
原始光谱矩阵 X → 标准化 → PCA → 保留约 5 个主成分 → 每条光谱的 PCA 系数
```

PCA 的作用：
1. 降低输出维度，减少模型直接预测 41 个波长点的难度；
2. 保留主要光谱变化趋势；
3. 用解释方差图说明主成分数量选择依据；
4. 结合课程中学习过的降维知识。

### 8.3 机器学习模型设计

本项目将光谱估计任务转化为回归任务：

- 输入：天气、时间、地点、太阳位置、室外照度等环境特征
- 输出：光谱 PCA 主成分系数

对比以下五个模型（全部 sklearn 标准实现）：

| 模型 | 作用 | 选择原因 |
|------|------|----------|
| Linear Regression | 线性基准模型 | 简单，作为对照组 |
| KNeighborsRegressor | KNN 回归 | 体现课程中 KNN 思想 |
| DecisionTreeRegressor | 决策树回归 | 结构直观，便于解释 |
| RandomForestRegressor | 随机森林回归 | 稳定性较好，适合非线性关系 |
| MLPRegressor | 多层感知机 | 体现神经网络基础，作为扩展模型参与对比 |

五个模型使用相同训练集、测试集、输入特征和 PCA 输出目标。最终按测试集统一指标选择最佳模型，不预设随机森林或 MLP 一定最好。CNN 不加入主实验：当前输入是表格型环境特征，CNN 更适合图像或序列数据，写入改进方向。

**为什么使用机器学习模型而不是直接使用大模型预测？**

LLM 适合辅助理解、资料整理、报告润色和代码解释，但不适合直接承担数值预测核心。自然光光谱估计是连续数值回归问题，需要稳定、可复现、可评价的模型。自己训练机器学习模型的价值包括：
- 准确性：可以用 MAE、RMSE、R² 等指标评价结果
- 可控性：模型输入、输出、训练数据和参数都可检查
- 可复现性：同样数据和代码可以重复运行
- 隐私与部署：未来如果接入室内传感器或用户位置数据，本地模型更容易保护隐私
- 效率：传统机器学习模型推理速度快，适合实时照明调节
- 课程适配：能体现 Python、PCA、sklearn、回归模型、多模型对比等课程内容

大模型参与方式主要是过程实施辅助，例如辅助查找资料、理解概念、检查代码注释和报告语言、解释模型结果，不替代核心数据处理、模型训练和结果评价。

### 8.4 模型评价指标

| 指标 | 含义 |
|------|------|
| MAE | 平均绝对误差，反映平均预测偏差 |
| RMSE | 均方根误差，对较大误差更敏感 |
| R² | 决定系数，反映模型解释能力 |
| 训练时间 / 预测时间 | 反映模型成本 |
| 光谱曲线对比 | 通过图像直观展示预测效果 |

最终展示方式：模型评价指标表、五模型误差对比柱状图、预测光谱与实测光谱曲线对比图。

### 8.5 光谱还原

模型预测出的不是原始 41 维光谱，而是 PCA 主成分系数。预测完成后，使用 PCA 的 inverse_transform() 方法还原：

```
模型预测 PCA 系数 → PCA 逆变换 → 还原为 380nm～780nm 相对光谱曲线 → 与实测光谱对比
```

### 8.6 室内多通道 LED 照明补偿设计

室内灯具假设为七通道光谱可调 LED（深蓝/蓝光、青光、绿光、琥珀光、红光、暖白、冷白），通道光谱全部来自公开实测 LED 数据。

补偿思路：

目标光谱 - 当前预测自然光贡献 = 需要人工补偿的光谱

然后通过非负最小二乘计算各 LED 通道的推荐比例，使"自然光 + 人工补偿光"尽量接近目标光谱。输出包括当前预测自然光谱、目标光谱、推荐通道比例、补偿后光谱曲线和补偿前后误差对比。

同时实现传统双色温 LED 对照方案：混光只有冷暖配比和亮度两个自由度，在同一规则下求最优解，用来说明传统方案能调冷暖但光谱形状受限，多通道补偿可以更细地贴近目标光谱。补偿结果还会通过 CIE 2015 标准观察者色匹配函数（CIE 2006 生理学数据推导，刻意不用 1931 年旧观察者）换算成屏幕近似颜色，以统一大小、统一亮度归一化的圆形光斑展示色度差异，并注明屏幕颜色只是近似色度预览。

**行业前沿说明：**
高质量照明正在从亮度/色温控制走向光谱控制。多通道 LED 混光可以比 RGB 更细致地调节光谱，避免窄峰光谱造成的显色问题。课程设计中使用七通道模型，是为了表达"光谱级照明补偿"的思路，没有做真实硬件控制。

4. PCA 为什么适合处理光谱数据；
5. 五个模型如何统一比较，最终为什么选择某个模型；
6. 补偿设计如何从预测结果延伸到应用场景；
7. 项目有哪些不足，未来如何改进。

### 10.2 报告中的谨慎表述

为了避免夸大，报告中使用以下表述：

本项目使用公开实测天光光谱数据作为主要数据来源，并结合数据集元数据与历史天气特征构建模型输入。天气数据为按地点和时间对齐的历史天气特征，不等同于现场同步气象测量。

本项目关注的是自然光"相对光谱形状"的估计，不做绝对辐照度的精确物理建模，模型估计也不能替代现场光谱仪实测。

本项目中的 LED 通道补偿为算法演示方案，未接入真实硬件系统，后续可结合智能灯具和光谱传感器进一步验证。

### 11.2 答辩表达重点

答辩时围绕下面这条线讲：

我们为什么做这个题？因为人长期在室内，而照明质量对室内体验很重要。

为什么关注光谱？因为亮度和色温不能完整描述光照，同色温、同亮度下光谱可能不同。

我们怎么做？用公开实测天光光谱数据和按地点时间对齐的历史天气特征构建数据集，用 PCA 降维，再用机器学习估计光谱主成分。

怎么证明模型有效？五个模型在相同条件下对比，用 MAE、RMSE、R²、训练和预测时间统一评价，由真实结果选择最佳模型。

有什么应用？根据预测自然光谱，计算七通道 LED 补偿比例，使室内合成光谱更接近目标自然光谱。

项目有什么不足？天气特征与光谱测量不一定完全同步；模型估计不能替代现场实测；样本集中在单一站点；补偿方案没有接入硬件。

## 十七、项目特色

1. **真实公开光谱数据支撑**：数据下载、清洗、合并全部由 Python 代码完成，可以复现；
2. **输入端低成本**：应用时只需要天气、时间、地点、太阳位置等容易获得的特征；
3. **PCA 降维表示高维光谱**：41 维光谱压缩为少量主成分，有解释方差依据；
4. **多模型真实对比**：五个模型统一指标评价，结论由数据决定；
5. **预测结果进入照明补偿应用闭环**：从环境特征一直走到 LED 通道比例和误差改善；
6. **具有一定前瞻性**：关注从"照度照明"走向"光谱照明"的方向。

## 十八、项目价值

**现实价值**：人长期在室内，照明质量影响学习、工作、视觉舒适和生活体验；传统照明往往只关注亮度、色温，不能完整描述光照质量。

**技术价值**：把真实光谱数据引入课程设计；用 PCA 处理高维光谱数据；用机器学习把天气、时间、太阳位置和光谱联系起来。

**应用价值**：未来可用于智能照明、健康照明、学习和办公空间照明；可与天气 API、室内传感器、智能灯具结合；可作为后续真实光谱采集与硬件控制的原型。

**教学价值**：覆盖 Python 数据处理、可视化、机器学习、模型评价和应用设计；项目结果有图、有表、有模型、有页面，适合代码演示和 PPT 答辩。

## 十九、项目不足与改进方向

### 19.1 不足
1. 天气特征为按地点和时间对齐的历史数据，与光谱测量不一定完全同步；
2. 有效样本几乎全部来自单一站点，跨地区泛化能力有限；
3. 模型估计不能替代现场光谱仪实测；
4. 光谱补偿算法较简单，未考虑真实房间窗户、反射、遮挡和灯具硬件差异；
5. 未接入真实智能灯具或硬件传感器。

### 19.2 改进方向
1. 寻找或采集更多地区（尤其是中国）的实测天光光谱数据；
2. 接入实时天气 API，获取云量、湿度、能见度等实时数据；
3. 加入廉价照度传感器或色温传感器，对预测结果进行实时校正；
4. 结合智能灯具，实现实际 LED 通道控制；
5. 若输入扩展到天空图像或时间序列，可尝试 CNN 等更适合此类数据的模型；
6. 进一步研究不同光谱对视觉舒适度、节律健康和颜色还原的影响。
